MCE

Monadic Context Engineering
A composable framework for AI agent orchestration built on monad transformers, explicit state, deterministic errors, and applicative parallelism.

Yifan Zhang
Princeton University
StateT + EitherT + IO Async Applicatives Explicit Effects Composable Agents Deterministic Failures

Abstract

Large language model agents often rely on imperative orchestration that is difficult to test, scale, or reason about. MCE introduces a formal, algebraic foundation based on Functors, Applicatives, and Monads, enabling a uniform structure for stateful, fallible, and asynchronous workflows. By stacking monad transformers, MCE composes these concerns into a single, reusable interface for agent development.

The result is a workflow model that reads like a specification while remaining runnable, observable, and easy to extend with tool effects. Context, errors, and side effects are explicit at the type level, so orchestration logic stays predictable as systems scale.

Monad Stack

MCE models every run as a transformer stack that composes state, error handling, and effects into a single algebra. Each layer is swappable, so you can change persistence, error types, or execution backends without rewriting workflows.

StateT

Context as a first-class value

Thread agent memory, tool outputs, and intermediate plans without manual plumbing.

EitherT

Deterministic failures

Short-circuit errors with structured context and predictable recovery semantics.

Task / IO

Explicit effects

Encapsulate tool calls and async work so every side effect is testable and observable.

Declarative Flow

The AgentMonad exposes a small surface area: then for sequencing, map for pure transforms, and gather for parallel branches. Workflows stay linear and readable while retaining full introspection. 

// The workflow reads like a spec.
flow = AgentMonad.start(initial_state)
  .then(plan_action)
  .then(execute_tool)
  .then(synthesize_answer)
  .then(format_output)

Applicative Parallelism

Independent tasks can be run concurrently with AsyncAgentMonad.gather. The result preserves ordering and merges shared state with a configurable strategy, while keeping error handling deterministic. 

gathered = AsyncAgentMonad.gather([
  news_flow,
  weather_flow,
  stocks_flow,
])
result = await gathered.run()

Meta-Agents

Meta-agents operate on workflows instead of raw data. They synthesize sub-agent chains, run them, and combine results into a higher-level monadic context. MCE keeps these systems predictable by reusing the same algebra at every layer.

Planner

Compose subflows

Build new agent graphs from reusable primitives and apply policy constraints.

Evaluator

Score and repair

Validate outputs, reroute failures, and produce structured feedback loops.

Governor

Control tools

Gate expensive calls, enforce budgets, and log traces with minimal coupling.
